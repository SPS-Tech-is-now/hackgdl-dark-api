API1 BOLA — Broken Object Level Authorization

DEMO EN VIVO
$ curl http://crapi-web/identity/api/v2/vehicle/VICTIM_UUID/location \
     -H "Authorization: Bearer $ATTACKER_TOKEN"

HTTP 200 OK
{"carId":"88ee25b0","lat":20.6534,"long":-103.3465,"vehicleLocation":"Guadalajara, Jalisco"}

¿Por qué funciona?

  • El API no valida si el vehicleId pertenece al usuario autenticado
  • Solo verifica que el JWT sea válido
  • Los IDs son predecibles / obtenibles de otros endpoints

Impacto

  • Tracking de ubicación GPS de víctimas
  • GDPR / LFPDPPP violation
  • CVSS: 8.1 (High)
  • Explotable sin conocimiento previo

// OpenZiti / NetFoundry

¿Cómo funciona un Dark Service?

Principios

  • Never trust, always verify
  • El servicio hace conexión saliente al broker
  • No abre puertos de entrada
  • No tiene registro DNS público
  • Mutual TLS + certificados por identidad
  • Autorización granular por servicio
ziti@darkservice — shell
# Tu API registrada como dark service
$ ziti edge create service mi-api
Created service: mi-api

# La API conecta SALIENTE al broker
# (no abre ningún puerto de entrada)

$ ziti tunnel bind --identity server.json \
  --service mi-api \
  --host localhost --port 8888
Tunneling mi-api → localhost:8888

# El cliente necesita identidad válida
$ ziti tunnel run client.json
Intercepting: dark.api.internal → ZT
OpenZiti es open source. NetFoundry es el SaaS administrado. Ambos soportan Node.js, Python, Go, Java — y MCP para agentes de IA.

// Comparativa

API Pública vs. Dark Service

Técnica Ofensiva API Pública Dark Service
nmap port scan ✅ Puertos visibles ✗ 0 hosts up
Endpoint enumeration (ffuf) ✅ Endpoints mapeados ✗ Connection refused
BOLA / IDOR ✅ Datos de víctimas ✗ Sin acceso
Brute force login ✅ Sin rate limiting ✗ Sin acceso
Data exposure ✅ PII en respuestas ✗ Sin acceso
DNS lookup ✅ IP pública ✗ NXDOMAIN
TLS fingerprinting ✅ Cert info ✗ Sin certificado expuesto
El código del API tiene las mismas vulnerabilidades en ambos casos.
La diferencia es la arquitectura de red.
// Implementación

Hazlo tú mismo — OpenZiti en 4 pasos

# 1. Levantar controlador (docker o NetFoundry SaaS)
$ docker run -d openziti/quickstart

# 2. Registrar tu API como dark service
$ ziti edge create service "mi-api" --role-attributes api-service

# 3. SDK en tu app — solo cambiar cómo escucha el servidor
# (Node.js / Python / Go / Java)
$ npm install @openziti/ziti-sdk-nodejs

# 4. Crear identidades con certificados para clientes
$ ziti edge create identity device "mi-cliente" \
    --role-attributes api-clients -o client.jwt
